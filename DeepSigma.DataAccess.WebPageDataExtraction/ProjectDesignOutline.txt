What to use in C#

There are three common levels.

1. Plain HTML fetch + parse

Good when pages are mostly server-rendered.

Use:

HttpClient to download HTML
Html Agility Pack if you want tolerant HTML parsing and XPath support
AngleSharp if you want a more browser-like DOM and CSS selectors
2. Main-content extraction

Good when you want article/body text instead of raw HTML.

A useful option is SmartReader, which is a .NET library for extracting the main content of a web page, based on Mozilla Readability.

This is usually much better than just grabbing all <p> tags, because it tries to remove nav bars, ads, footers, and sidebars.

3. Browser automation

Needed when the page is rendered by JavaScript.

Use Playwright for .NET. It supports Chromium, WebKit, and Firefox, and lets you load the page after scripts run.

Basic approach in C#
A. Fetch the HTML
public async Task<string> DownloadHtmlAsync(string url, CancellationToken ct = default)
{
    using var http = new HttpClient();
    http.DefaultRequestHeaders.UserAgent.ParseAdd(
        "MySearchBot/1.0 (+https://yourdomain.example/bot)");

    using var response = await http.GetAsync(url, ct);
    response.EnsureSuccessStatusCode();

    return await response.Content.ReadAsStringAsync(ct);
}
B. Parse and extract text with AngleSharp
using AngleSharp;
using AngleSharp.Dom;

public async Task<string> ExtractTextAsync(string html)
{
    var config = Configuration.Default;
    var context = BrowsingContext.New(config);
    var document = await context.OpenAsync(req => req.Content(html));

    // crude fallback extraction
    var paragraphs = document.QuerySelectorAll("article p, main p, p");
    var text = string.Join("\n\n",
        paragraphs.Select(p => p.TextContent.Trim())
                  .Where(t => !string.IsNullOrWhiteSpace(t)));

    return text;
}

This is a decent fallback, but not the best “production quality” article extractor.

C. If the page is JS-heavy, use Playwright
using Microsoft.Playwright;

public async Task<string> DownloadRenderedHtmlAsync(string url)
{
    using var playwright = await Playwright.CreateAsync();
    await using var browser = await playwright.Chromium.LaunchAsync(
        new BrowserTypeLaunchOptions { Headless = true });

    var page = await browser.NewPageAsync();
    await page.GotoAsync(url, new PageGotoOptions
    {
        WaitUntil = WaitUntilState.NetworkIdle
    });

    return await page.ContentAsync();
}

Then feed that HTML into your extractor.

Best production design

I would separate this into components:

public interface IWebPageFetcher
{
    Task<WebPageFetchResult> FetchAsync(string url, CancellationToken ct = default);
}

public interface IContentExtractor
{
    Task<ExtractedContent> ExtractAsync(WebPageFetchResult page, CancellationToken ct = default);
}
IWebPageFetcher

Responsible for:

HttpClient
redirects
content-type checks
gzip/brotli
timeouts
retry policy
optional Playwright fallback
IContentExtractor

Responsible for:

parse HTML
strip scripts/styles/nav/footer
extract title
extract main article/body
normalize whitespace
return clean text
What to return

Do not just return a string. Return structured data:

public sealed record ExtractedContent(
    string Url,
    string? Title,
    string? Byline,
    string? Excerpt,
    string MainText,
    string? Language,
    DateTimeOffset? PublishedAt);

That makes downstream ranking and indexing much easier.

The practical rules you want

Before extracting content, check:

Content-Type is HTML
response size is not huge
page is not blocked / login-only
respect robots.txt if you are crawling at scale

For robots handling in .NET, there are libraries like Robots Exclusion Tools and other robots.txt parsers for .NET.
